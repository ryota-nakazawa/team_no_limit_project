# フェーズ3: 利用イメージ整理

## 想定ユーザー
- 主利用者: 日々の判断と調整が多い課長
- 補助利用者: 課長を支援する主任、係長、部門アシスタント

## 利用シーン
1. 朝の業務整理
   - 今日の会議、依頼、未完了タスクを整理し、優先順位を出す
2. 会議前準備
   - 目的、参加者、決めることからアジェンダと論点を作る
3. 会議後フォロー
   - 決定事項、宿題、連絡文面をまとめる
4. メンバー支援
   - 1on1 の観点整理、フィードバック草案、相談整理
5. 上申/報告
   - 上司向け報告、他部署調整文、ステークホルダー説明文を作る

## 役割分担
- ユーザー: 状況、論点、制約などの一次情報を入力する
- アプリ: 入口案内、入力観点提示、たたき台生成、抜け漏れ確認を行う

## 画面と操作の流れ
1. ホームで業務カテゴリを選ぶ
2. カテゴリ詳細で「何に使えるか」「必要入力」「出力イメージ」を確認する
3. サンプル指示を選ぶか自由入力する
4. チャット結果を見て、追加条件を与えて磨く

## PoC 成立条件
- カテゴリ選択だけで用途が理解できる
- 代表ユースケースごとに初回の相談文を迷わず始められる
- 出力がそのまま下書きや論点整理に使える水準に近い

## PoC 対象外
- 権限管理
- 実データの自動収集
- 承認ワークフロー自動化
