# フェーズ4: 入出力整理

## カテゴリ別入力/出力

### 1. 日次整理
- 入力: 今日の予定、未完了タスク、気になる案件、締切
- 出力: 優先順位付きタスクリスト、注意点、最初の 3 アクション

### 2. 会議運営
- 入力: 会議目的、参加者、決めたいこと、現状論点、制約
- 出力: アジェンダ、確認質問、議事メモ骨子、会議後フォロー文面

### 3. メンバー支援
- 入力: 対象者、状況、期待行動、懸念点、伝え方のトーン
- 出力: 1on1 観点、フィードバック下書き、相談整理メモ

### 4. 報連相・調整
- 入力: 相手、背景、伝えたい要点、依頼内容、期限
- 出力: 報告文、依頼文、調整メッセージ、要約

### 5. 計画・分析
- 入力: KPI、現状値、目標、課題仮説、検討中の打ち手
- 出力: 課題整理、原因仮説、次アクション、上申用要点

## 表示単位
- カテゴリ単位で入口を見せる
- 各カテゴリ内で用途、入力観点、想定出力をカード表示する
- 生成結果は会話単位で保持する

## データの流れ
1. ユーザーがカテゴリを選ぶ
2. カテゴリに紐づく説明とテンプレートを UI に表示する
3. 入力テキストを送信する
4. カテゴリ別システムプロンプトを付与して応答を生成する
5. 会話履歴として画面に表示する

## ファイル/フォーム/生成物の関係
- フォーム: 自由入力欄、将来的にはカテゴリ別入力フォームへ拡張
- 生成物: 画面内チャットの出力文
- ファイル: 現段階では仕様書 Markdown を整備し、生成物の保存先は未対応
