# 課長お助けエージェント プロジェクト整理

## 1. 背景
- 課長は会議、報連相、メンバー支援、業績確認、資料作成などの業務が断続的に発生し、判断とコミュニケーションの切り替え負荷が高い。
- 既存アプリは音声チャットを中心としたデモ構成であり、課長業務の入口や用途別導線が明確ではない。
- 業務カテゴリごとに相談の切り口を用意し、迷わず使い始められる UI と最小限の生成支援を備えたアプリへ再整理したい。

## 2. 現状業務
- 日次運営: 優先順位整理、進捗確認、未処理タスク抽出
- 会議運営: アジェンダ作成、論点整理、議事メモ、会議後フォロー
- メンバー支援: 1on1 準備、フィードバック整理、相談対応
- 報連相/調整: 上司報告、部門間調整、顧客/関係者向け文面作成
- 計画/分析: KPI 確認、課題整理、次アクション設計

## 3. 現状課題
- どの用途で使えるアプリかが初見で分かりにくい。
- プロンプトが旧用途向けで、課長業務に即していない。
- 入力項目や出力物の粒度が未定義で、PoC 成否の判断基準がない。
- 既存画面は会話 UI 中心で、カテゴリナビゲーションや利用シーン説明が弱い。

## 4. 期待効果
- 課長が用途別に迷わず起動し、短時間で下書きや論点整理を得られる。
- 会議準備、日次整理、報告文面作成の着手時間を短縮できる。
- 出力の型をそろえ、レビューや横展開がしやすくなる。

## 5. 制約
- 初期段階は既存 React アプリを活用し、PoC として最小構成で成立させる。
- 外部システム連携は前提にせず、まずは手入力ベースで使えることを優先する。
- 現時点では課長本人の詳細な業務ヒアリング結果がないため、一般的な管理職業務をたたき台として整理する。

## 7. フェーズ1の判断
- まずは「課長業務のカテゴリが見える」「カテゴリ別に相談開始できる」「最低限の出力型が見える」の 3 点を PoC の起点にする。
