# フェーズ5: 要件定義書

## 1. 目的
- 課長が日々の管理業務を短時間で整理し、下書きや論点整理を得られる「課長お助けエージェント」を提供する。

## 2. 対象業務
- 日次整理
- 会議運営
- メンバー支援
- 報連相・調整
- 計画・分析

## 3. 対象ユーザー
- 主対象: 現場マネジメントを担う課長

## 4. 機能要件
- 業務カテゴリをナビゲーションで一覧表示できること
- 選択カテゴリごとに用途説明、入力観点、想定出力を表示できること
- カテゴリごとのシステムプロンプトで会話支援できること
- 代表ユースケースのクイックスタート文を提示できること
- 生成結果を会話履歴として閲覧できること

## 5. 非機能要件
- PC での利用を主対象にしつつ、モバイルでも閲覧可能なレスポンシブ UI とする
- 初見で用途が把握できる情報設計にする
- PoC 段階では手入力で完結することを優先する

## 6. 入力
- 自由入力テキスト
- 将来拡張用の観点: 日付、対象者、会議名、KPI、期限

## 7. 出力
- 優先順位整理
- アジェンダ/議事メモ骨子
- フィードバック草案
- 報告/依頼文面
- 課題整理と次アクション

## 8. 制約
- 既存 React アプリをベースに PoC を実装する
- 外部 SaaS 連携は次段階に回す
- 詳細業務ヒアリング未了のため、初期カテゴリは仮説ベースで構成する
